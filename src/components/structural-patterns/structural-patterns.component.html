Adapter
  • Definición: Permite la colaboración entre clases con interfaces incompatibles mediante una clase adaptadora.
  • Problema: Integrar componentes antiguos o de terceros.
  • Ventajas: Reutilización sin modificar código original.
  • Desventajas: Puede volverse complejo si se abusa.
  • Usos: Librerías externas, APIs antiguas.
  • Patrones relacionados: Bridge, Decorator.
Bridge
  • Definición: : Desacopla una abstracción de su implementación, permitiendo variar ambas de forma independiente.
  • Usos: Interfaces gráficas multiplataforma.
  • Ventajas: Escalabilidad, evita herencia múltiple.
  • Desventajas: Diseño inicial más complejo.
  • Patrones relacionados: Adapter, Strategy.
Composite
  • Definición: Permite tratar objetos individuales y grupos de objetos de forma uniforme.
  • Usos: Representar jerarquías (menús, carpetas).
  • Ventajas: Operaciones recursivas simples.
  • Desventajas: Puede ser difícil restringir componentes.
  • Patrones relacionados: Iterator, Visitor.
Decorator
  • Definición: Agrega responsabilidades a un objeto de manera dinámica sin modificar su clase.
  • Ventajas: Más flexible que la herencia.
  • Desventajas: Difícil rastrear comportamiento con muchos decoradores.
  • Usos: Streams en Java, procesamiento de texto.
  • Patrones relacionados: Composite, Strategy.
Facade
  • Definición: Proporciona una interfaz simplificada a un sistema complejo de clases.
  • Ventajas: Facilita el uso de subsistemas.
  • Desventajas: Puede ocultar detalles importantes.
  • Usos: Librerías o subsistemas complicados.
  • Patrones relacionados: Mediator, Singleton.
Flyweight
  • Definición: Minimiza el uso de memoria compartiendo datos entre objetos similares.
  • Ejemplo: Caracteres de un procesador de texto.
  • Ventajas: Optimización de recursos.
  • Desventajas: Mayor complejidad para diferenciar estados internos.
  • Usos: Juegos, editores de texto.
  • Patrones relacionados: Prototype, Singleton.
Proxy
  • Definición: Proporciona un objeto sustituto que controla el acceso a otro objeto.
  • Tipos: Virtual, remoto, de seguridad.
  • Ventajas: Control de acceso, inicialización diferida.
  • Desventajas: Añade niveles de indirección.
  • Usos: Servicios remotos, seguridad, cache.
  • Patrones relacionados: Decorator, Adapter.
