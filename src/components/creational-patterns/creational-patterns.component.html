Singleton
  • Definición: Garantiza que una clase tenga una única instancia y proporciona un punto de acceso global a ella.
  • Problema que resuelve: Evita la creación múltiple de objetos que representan recursos compartidos como conexiones de base de datos.
  • Ventajas: Control centralizado, bajo uso de memoria.
  • Desventajas: Dificulta pruebas unitarias; puede introducir acoplamientos fuertes.
  • Usos: Sistemas de logging, manejo de configuraciones globales, drivers.
  • Patrones relacionados: Factory Method, Facade.
Factory Method
  • Definición: Define una interfaz para crear un objeto, pero permite que las subclases decidan qué clase instanciar.
  • Utilidad: Facilita la extensibilidad sin conocer las clases exactas a crear.
  • Ventajas: Desacopla la lógica de creación del cliente.
  • Desventajas: Aumenta la complejidad con múltiples subclases.
  • Usos: Creación de componentes UI, controladores web.
  • Patrones relacionados: Abstract Factory, Template Method.
Abstract Factory
  • Definición: : Permite crear familias de objetos relacionados sin especificar sus clases concretas.
  • Problema que resuelve: Mantiene la coherencia entre productos que deben funcionar juntos.
  • Ventajas: Asegura compatibilidad entre objetos.
  • Desventajas: Difícil agregar nuevas familias de productos.
  • Usos: Diseño de interfaces gráficas multiplataforma, sistemas de temas o "skins".
  • Patrones relacionados: Factory Method, Builder.
Builder
  • Definición: Separa la construcción de un objeto complejo de su representación final.
  • Utilidad: Útil cuando un objeto puede tener múltiples representaciones o configuraciones.
  • Ventajas: Control detallado del proceso de creación.
  • Desventajas: Sobrecarga si el objeto no es complejo.
  • Usos: Creación de documentos, mensajes complejos, configuradores de productos.
  • Patrones relacionados: Composite, Fluent Interface
Prototype
  • Definición: Permite crear nuevos objetos copiando una instancia prototipo
  • Ventajas: Evita la creación repetida desde cero.
  • Desventajas: Puede ser difícil clonar objetos con referencias complejas.
  • Usos: Juegos (clonación de personajes), editores gráficos.
  • Patrones relacionados: Singleton, Memento.
